
# Module 9: Social Engineering

## 1. Social Engineering Concepts

**Definition:**  
Psychological manipulation of people to reveal confidential information or perform actions.

**Key Principle:**  
**Humans are the weakest link** in cybersecurity.

**Attack Cycle:**  
1. **Research** → Social media, websites  
2. **Hook** → Establish trust  
3. **Play** → Extract info/action  
4. **Exit** → Disappear  

**Psychological Triggers:**  
- **Authority** (“IT department”)  
- **Urgency** (“Account locked!”)  
- **Fear** (“Legal action”)  
- **Greed** (“You won!”)  
- **Curiosity** (“Confidential list”)  

---

## 2. Human-Based Techniques

| Technique | Description | Example |
|-----------|-------------|---------|
| **Impersonation** | Pretend to be trusted person | Fake IT support call |
| **Shoulder Surfing** | Watch user input | Password viewing |
| **Dumpster Diving** | Trash for sensitive info | Printed passwords |
| **Tailgating** | Follow authorized person | “Hold door” |
| **Vishing** | Phone phishing | Fake bank OTP call |
| **Smishing** | SMS phishing | Verify account link |
| **Elicitation** | Casual info extraction | Conversation probing |

---

## 3. Computer-Based Techniques

1. **Phishing**  
   **Definition:** Fraudulent emails stealing credentials.  
   **Types:**  
   - **Spear phishing** → Targeted  
   - **Whaling** → Executives  

2. **Pharming**  
   **Definition:** Redirect to fake sites (DNS/hosts file).  

3. **Malicious Attachments**  
   - Exploit PDFs  
   - Word macros  

4. **Pop-up Attacks**  
   - Fake infection alerts  

5. **Fake Software/Updates**  
   - Malicious antivirus  

6. **Watering Hole**  
   - Compromise industry websites  

---

## 4. Mobile-Based Techniques

1. **Smishing** (SMS phishing)  
2. **Malicious Apps** (fake banking)  
3. **Bluetooth Attacks**  
   - **Bluejacking** → Spam  
   - **Bluesnarfing** → Data theft  
4. **QR Code Attacks** (malicious links)  
5. **Fake Mobile Networks** (rogue base stations)  

---

## 5. Countermeasures

**Technical:**  
```
• Email filtering
• Anti-phishing tools
• Endpoint protection
• MFA
```

**Human:**  
```
• Security awareness training
• Phishing simulations
• Identity verification policy
```

**Best Practices:**  
```
• Never share credentials
• Verify before sharing info
• Avoid unknown links/attachments
• Strong authentication
```
